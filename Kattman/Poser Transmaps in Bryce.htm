10 Step Poser to Bryce
Transparency Map Tutorial

I am using Koz's fantastic Alice Hair for this tutorial. 
This image shows how great it looks inside poser itself,
yet when you load it into Bryce, it can look really bad. 
This tutorial will show you how to get this same look in Bryce.
The first thing we need to do so Bryce can use the file is to export the mesh to OBJ format.  These are the settings I use for all my exports to Bryce. 
It gives me a ton of grouped objects to play with
so I tend to export each character separately.
On first importing to Bryce I created a render. 
Bryce of course asked me for the location of the texture map but ignored the transmap. 
As you can see, it is textured nicely but the edges are real hard,
not quite good enough for hair.
Select your object and go into the material editor. 
Once there, look at the transparency slider, notice the dot. 
Place that dot in an empty channel.  That channel will load a default texture. 
Change it to a picture texture then go into the deep texture editor for that channel. 
What you will see are three large boxes near the top and
a bunch of little boxes at the bottom.  Select an empty box and load the transmap there.
Once the transmap is loaded you will see it fill the top boxes with the selection. 
Keep in mind that the Bryce editor will show a
checkerboard pattern for all transparent areas. 
Looking at the image to the right, you can see that in the third box
there is no transparent areas.  The third box is the final map used. 
The first box is basically a starting point, the second is a modifier. 
We need to use both of these to create the transparency.
Bryce tends to read transmaps backwards so we have to flip the values around here. 
The method of doing so is fairly easy. 
In the first box select copy, then paste into the second box. 
Bryce will ask if you want to delete, go ahead and do so. 
Next push that little black and white split button to reverse the colors. 
Copy the second box back to the first, once again deleting when it asks. 
Next, push that black and white button again to reverse the second box one more time.  You should end up with something like the photo. 
Notice now how the third box shows the transparent area in checkerboard
and the rest as solid black. 
This is what we need so hit the check mark at the bottom of the editor to save it.
Upon returning to the material editor, these are the setting we should usually see. 
Notice the Surface diffusion is set to the first texture channel. 
That channel holds the texture map. 
The transparency slider is in the second channel. 
That channel holds the transparency map we just modified. 
It all looks good here so let's go render.
Well, the transparency went in, sorta. 
We can see the different strands of hair now, but if you look closely
you will see the transparent areas are ghosted. 
They have a white glow to them. 
This is caused by the diffusion on the texture map. 
Those parts of the texture map are white so it shows up here as a glow. 
We need to get rid of that, but how.
I had tried tweaking the diffusion slider changing channels
and all kinds of things to get rid of that ghosting effect. 
The solution became real simple once I found it. 
Notice the Diffusion slider, you will be tempted to set that to
the same channel as the surface diffusion hut don't. 
Leave the slider at 100 and set the dot to your transparency channel. 
What this basically does is tell it to diffuse only those areas
not marked as transparent, essentially making all transparent areas truly transparent.
Go back out and render. 
You will see the fine hair as Koz wanted it to be seen. 
The ghosting is gone and the fringe looks good. 
The edges are nice and soft like hair should look. 
This same technique can be applied to bump maps but
you won't need to worry about the ghosting effect there
so setting the diffusion to the transparency channel isn't required. 
Unless of course you have a transmap applied also.
